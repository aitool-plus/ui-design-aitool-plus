## 常见问题与修复方案

### 问题1：风格偏差

**现象**：整体视觉与目标OS风格不符

**修复方案**：
1. 对照规范文件检查色彩/字体/间距参数
2. 修正Token引用，确保使用目标OS的原始值
3. 检查组件形态是否符合目标OS设计规范
4. 重新应用目标OS的圆角/阴影/动效特征

### 问题2：组件不一致

**现象**：同类组件在不同页面表现不一致

**修复方案**：
1. 统一Token引用，确保所有组件使用相同变量
2. 检查是否有硬编码样式覆盖Token
3. 建立组件样式复用机制
4. 进行全局样式审查

### 问题3：响应式问题

**现象**：特定尺寸下布局错乱

**修复方案**：
1. 检查断点设置是否正确
2. 验证各断点下的布局规则
3. 检查内容溢出处理
4. 测试极端尺寸下的表现

### 问题4：深色模式问题

**现象**：深色模式下颜色显示异常

**修复方案**：
1. 检查深色模式Token覆盖是否完整
2. 验证所有颜色变量都有深色对应值
3. 检查图片/图标是否适配深色模式
4. 测试对比度是否满足可读性要求

## 迭代流程

### 步骤1：问题定位

1. 对照还原度检查清单逐项检查
2. 记录所有不符合项
3. 按严重程度分类（阻塞/严重/一般/建议）
4. 确定问题根因

### 步骤2：方案设计

1. 针对每个问题制定修复方案
2. 评估修复影响范围
3. 确定修复优先级
4. 制定修复计划

### 步骤3：局部修改

1. 按优先级逐个修复问题
2. 每次修改后进行局部验证
3. 确保修改不引入新问题
4. 记录修改内容

### 步骤4：全量验证

1. 完成所有修改后进行全量检查
2. 重新执行还原度检查清单
3. 进行多设备/多浏览器测试
4. 进行无障碍测试

### 步骤5：输出报告

1. 汇总所有修改内容
2. 记录验证结果
3. 输出迭代报告
4. 归档版本记录

## 多风格对比优化建议

### 对比维度

1. **视觉差异**：色彩/字体/间距/圆角的差异点
2. **交互差异**：手势/动画/反馈的差异点
3. **组件差异**：组件形态/状态的差异点
4. **布局差异**：栅格/边距/安全区域的差异点

### 优化策略

1. **提取共性**：找出各风格间的共同部分，建立基础层
2. **封装差异**：将风格差异封装为可配置项
3. **增量适配**：新增风格时基于基础层增量适配
4. **版本管理**：各风格独立版本管理，避免相互影响
