# 数据展示组件规范

## 1. 组件概述

数据展示组件用于将结构化或半结构化信息以可视化的方式呈现给用户，涵盖列表、卡片、标签/芯片、表格、头像五大核心组件。本规范确保数据在不同OS环境下的可读性、一致性与操作效率。

## 2. 通用规范

### 2.1 结构描述

所有数据展示组件遵循统一的结构分层：
- **容器层**：定义组件外边界、圆角、阴影或边框
- **内容层**：承载文本、图像、图标等核心信息
- **操作层**：提供交互入口（点击、滑动、长按等）
- **状态层**：通过颜色、透明度、遮罩反馈数据状态（选中、禁用、加载中）

### 2.2 尺寸参数

| 组件类型 | 最小尺寸 | 标准尺寸 | 最大尺寸 | 间距范围 |
|---------|---------|---------|---------|---------|
| 列表行高 | 44dp | 52dp | 64dp | 12-20dp |
| 卡片圆角 | 8dp | 12dp | 16dp | 12-20dp |
| 标签/芯片高度 | 24dp | 28dp | 36dp | 8-12dp |
| 表格行高 | 40dp | 48dp | 56dp | 0-1dp |
| 头像尺寸 | 24dp | 40dp | 64dp | 4-8dp |

### 2.3 状态变体

| 状态 | 视觉表现 | 适用组件 |
|-----|---------|---------|
| 默认 | 基础色值、常规背景 | 全部 |
| 悬停 | 背景色提亮5%、指针变化 | 列表、卡片、表格行 |
| 按下 | 背景色降低8%、缩放0.99 | 列表、卡片 |
| 选中 | 主色描边或主色背景+反白文字 | 列表、卡片、标签 |
| 禁用 | 透明度0.38、移除交互响应 | 全部 |
| 加载中 | 骨架屏或旋转指示器替代内容 | 列表、卡片、表格 |
| 空数据 | 空状态插画+提示文本 | 列表、表格、卡片组 |

### 2.4 无障碍

- 列表项必须支持屏幕阅读器顺序朗读，提供明确的语义化标签
- 表格必须支持表头与数据单元的关联声明（scope/headers）
- 头像必须提供alt文本或aria-label描述用户身份
- 标签/芯片必须支持键盘导航与选择
- 数据展示组件的颜色对比度不低于4.5:1（正文文本）或3:1（大号文本/图标）
- 动态加载的数据必须通知屏幕阅读器内容变化（aria-live）

## 4. 设计禁忌

1. **禁止列表行高低于44dp**：低于此值将导致触控困难，信息密度过高影响可读性
2. **禁止卡片阴影过重**：阴影扩散值不得超过12dp，透明度不得超过12%，避免视觉脏污
3. **禁止标签/芯片纯装饰使用**：标签必须承载可筛选、可删除或可点击的交互语义，禁止仅作静态展示
4. **禁止表格列宽不足导致文本截断**：表格必须提供列宽自适应或横向滚动机制，关键信息不可省略号隐藏
5. **禁止头像无默认占位**：头像加载失败或无图片时必须显示默认占位，禁止空白或破损图标
6. **禁止列表与卡片混用无规律**：同一页面内同类数据展示形式必须统一，禁止部分用列表部分用卡片
7. **禁止表格表头固定而数据不固定**：长表格必须同步固定表头与首列（如需要），或提供明确的滚动指示
8. **禁止标签颜色超过5种**：同一视图内标签颜色种类超过5种将破坏视觉层次，必须合并或降级处理
9. **禁止卡片内边距小于12dp**：卡片内容过于贴边将导致视觉拥挤，标准内边距为16dp
10. **禁止头像组重叠超过50%**：头像组重叠量不得超过头像直径的50%，否则无法辨识底层头像
