# 媒体组件规范

## 1. 组件概述

媒体组件用于展示图片、图标、插画、头像组及横幅等视觉内容，是界面信息传达的重要组成部分。本规范涵盖图片、头像组、图标、插画占位及横幅五大核心媒体组件，确保视觉内容在不同OS环境下的展示一致性与加载性能。

## 2. 通用规范

### 2.1 结构描述

所有媒体组件遵循统一的结构分层：
- **容器层**：定义媒体外边界、圆角、裁剪区域
- **内容层**：承载实际媒体资源（图片、图标、插画）
- **遮罩层**：提供渐变遮罩、颜色叠加、模糊效果
- **状态层**：加载中、加载失败、占位符等状态反馈

### 2.2 尺寸参数

| 组件类型 | 最小尺寸 | 标准尺寸 | 最大尺寸 | 圆角范围 |
|---------|---------|---------|---------|---------|
| 图片 | 40dp | 120dp | 全屏 | 0-16dp |
| 头像组 | 24dp/个 | 40dp/个 | 64dp/个 | 全圆角 |
| 图标 | 16dp | 24dp | 48dp | 0-4dp |
| 插画占位 | 120dp | 160dp | 240dp | 0-16dp |
| 横幅 | 120dp | 160dp | 200dp | 0-16dp |

### 2.3 状态变体

| 状态 | 视觉表现 | 适用组件 |
|-----|---------|---------|
| 默认 | 原始内容正常展示 | 全部 |
| 加载中 | 灰色占位或旋转指示器 | 图片、插画、横幅 |
| 加载失败 | 错误图标+重试提示 | 图片、插画、横幅 |
| 悬停 | 亮度变化、缩放或阴影增强 | 图片、头像组 |
| 按下 | 缩放0.98、亮度降低 | 图片、头像组、横幅 |
| 选中 | 主色边框或勾选标记 | 图片、头像组 |
| 禁用 | 灰度化、透明度0.5 | 图片、头像组、图标 |

### 2.4 无障碍

- 所有图片必须提供alt文本或aria-label，描述图片核心内容
- 头像必须提供用户名称或身份描述
- 图标必须提供语义化标签，不可仅作为装饰
- 插画占位必须提供相关操作说明或状态描述
- 横幅中的文字与背景对比度不低于4.5:1
- 动态媒体（自动轮播横幅）必须提供暂停控制

## 4. 设计禁忌

1. **禁止图片无圆角直接贴边**：图片在卡片或容器内必须保持与容器一致的圆角，禁止直角图片出现在圆角容器中
2. **禁止头像组重叠超过50%**：头像重叠量超过直径50%将导致底层头像无法辨识，标准重叠量为-8dp至-10dp
3. **禁止图标尺寸低于16dp**：过小的图标将无法辨识，标准最小尺寸为20dp
4. **禁止插画风格与OS不匹配**：iOS应用禁止使用Android Material风格的插画，必须保持系统视觉一致性
5. **禁止横幅文字覆盖在复杂背景上**：横幅背景图必须配合渐变遮罩（底部或全幅），确保文字可读性
6. **禁止图片加载无占位过渡**：图片加载必须有灰色占位或模糊缩略图过渡，禁止直接空白后跳变
7. **禁止图标使用彩色填充作为唯一语义**：图标的功能语义必须通过形状传达，颜色仅作辅助
8. **禁止头像组超过5个**：头像组超过5个将严重重叠且难以辨识，多余头像必须以+N形式提示
9. **禁止横幅高度超过200dp**：过高的横幅将占用过多屏幕空间，标准高度范围为120-160dp
10. **禁止媒体组件无错误降级**：所有媒体加载失败必须有明确的错误占位和重试机制，禁止显示破损图标或空白
