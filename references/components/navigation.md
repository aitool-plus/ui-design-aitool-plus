# 导航组件规范

> @author aitool.plus
> 本文档定义导航类组件的跨平台设计规范，覆盖9大OS风格差异对比。

---

## 一、顶部导航栏 (NavigationBar / TopAppBar)

### 1.1 组件概述

顶部导航栏是页面最上方的固定区域，承载页面标题、返回操作、功能入口等核心导航能力，是用户进入页面后第一视觉锚点。

### 1.2 通用规范

**结构描述：**
- 左侧区域：返回按钮（可选）、导航图标
- 中间区域：页面标题/搜索框
- 右侧区域：功能图标（1-3个）、溢出菜单

**尺寸参数：**
| 参数 | 数值 |
|------|------|
| 栏高度（手机） | 44-56dp |
| 栏高度（平板/桌面） | 48-64dp |
| 标题字号 | 16-20sp |
| 图标尺寸 | 24dp |
| 图标触控区域 | 48x48dp |
| 左右内边距 | 16dp |
| 图标间距 | 24dp |

**状态变体：**
| 状态 | 说明 |
|------|------|
| 默认 | 标准背景色+标题+图标 |
| 滚动收缩 | 内容滚动后收缩为仅图标模式 |
| 搜索模式 | 标题替换为搜索输入框 |
| 透明叠加 | 适用于图片顶部悬浮 |

**无障碍：**
- 所有图标按钮须提供 `contentDescription` / `aria-label`
- 标题文字对比度 >= 4.5:1
- 触控区域不小于 44x44dp（iOS）/ 48x48dp（Material）

### 1.4 设计禁忌

1. 顶部导航栏图标超过3个，功能入口过多应收纳至溢出菜单
2. 标题文字超过20字符且未做截断处理
3. 返回按钮触控区域小于44x44dp
4. 顶部导航栏高度在滚动过程中发生非预期的跳变
5. 透明模式下背景与标题文字对比度不足4.5:1

## 二、底部标签栏 (TabBar / NavigationBar)

### 2.1 组件概述

底部标签栏是固定在页面底部的导航组件，承载应用核心模块的快速切换能力，是移动端最核心的全局导航模式。

### 2.2 通用规范

**结构描述：**
- 标签项：图标+文字标签（2-5个）
- 可选角标（Badge）：数字/红点
- 可选中间凸起按钮（FAB样式）

**尺寸参数：**
| 参数 | 数值 |
|------|------|
| 栏高度（含安全区） | 49-80dp |
| 标签项宽度 | 均分屏幕宽度 |
| 图标尺寸 | 24dp |
| 文字字号 | 10-12sp |
| 图标与文字间距 | 4dp |
| 角标最小尺寸 | 16dp（红点8dp） |
| 选中指示器高度 | 2-3dp |

**状态变体：**
| 状态 | 说明 |
|------|------|
| 默认未选中 | 灰色图标+灰色文字 |
| 选中 | 主题色图标+主题色文字 |
| 禁用 | 30%透明度 |
| 角标提示 | 右上角红点或数字 |

**无障碍：**
- 每个标签项须设置 `selected` 状态语义
- 角标数字需通过 `accessibilityValue` 朗读
- 标签切换支持键盘方向键导航

### 2.4 设计禁忌

1. 标签项超过5个（移动端），超出应使用"更多"入口
2. 选中/未选中状态仅靠颜色区分，未提供图标变体差异
3. 角标数字超过999仍显示完整数字
4. 底部标签栏在横屏模式下未适配安全区域
5. iPad/macOS端使用底部标签栏而非侧边导航

## 三、侧边导航 (Drawer / SideNavigation)

### 3.1 组件概述

侧边导航是从屏幕侧边滑出的导航面板，承载完整的功能菜单与层级结构，适用于功能模块多、层级深的应用场景。

### 3.2 通用规范

**结构描述：**
- 头部区域：用户头像+昵称+描述（可选）
- 导航列表：图标+文字+角标
- 分组标题：分类标签
- 底部区域：设置/帮助等辅助入口

**尺寸参数：**
| 参数 | 数值 |
|------|------|
| 面板宽度（手机） | 280-320dp |
| 面板宽度（平板） | 240-400dp |
| 面板宽度（桌面） | 200-360dp(常驻) |
| 列表项高度 | 48-56dp |
| 图标尺寸 | 24dp |
| 图标与文字间距 | 16dp |
| 分组标题高度 | 48dp |
| 遮罩透明度 | 40%-60% |
| 滑出动效时长 | 250-350ms |

**状态变体：**
| 状态 | 说明 |
|------|------|
| 隐藏 | 不可见，通过手势/按钮唤出 |
| 展开覆盖 | 面板覆盖在内容上方 |
| 展开内推 | 面板推出，内容区压缩 |
| 常驻展开 | 平板/桌面端始终可见 |

**无障碍：**
- 遮罩区域须可被辅助技术识别为关闭按钮
- 列表项须支持键盘焦点导航
- 打开/关闭须通知 `accessibilityAnnouncement`

### 3.4 设计禁忌

1. 手机端侧边导航承载超过7个一级功能入口
2. 侧边导航层级超过3级嵌套
3. 侧边导航头部区域高度超过屏幕1/3
4. 常驻模式下侧边导航宽度占比超过屏幕40%（平板/桌面端）
5. 右滑手势与其他水平滚动操作冲突未处理

## 四、面包屑导航 (Breadcrumb)

### 4.1 组件概述

面包屑导航显示当前页面在层级结构中的位置路径，支持快速返回上级层级，适用于层级较深的内容浏览场景。

### 4.2 通用规范

**结构描述：**
- 路径项：文字链接
- 分隔符：/ 或 > 或 图标
- 当前页：不可点击的纯文字

**尺寸参数：**
| 参数 | 数值 |
|------|------|
| 文字字号 | 12-14sp |
| 路径项间距 | 8dp |
| 分隔符间距 | 8dp |
| 行高度 | 32-40dp |
| 折叠按钮尺寸 | 24dp |

**状态变体：**
| 状态 | 说明 |
|------|------|
| 完整展示 | 所有层级路径项可见 |
| 折叠展示 | 中间层级折叠为"..."按钮 |
| 悬停 | 路径项显示下划线 |

**无障碍：**
- 路径项须以 `nav` 语义包裹
- 分隔符须设置 `aria-hidden="true"`
- 当前页须设置 `aria-current="page"`

### 4.4 设计禁忌

1. 面包屑用于非层级结构的页面导航
2. 面包屑路径项超过5级且未做折叠处理
3. 当前页设置为可点击状态
4. 移动端首屏使用面包屑作为主要导航方式
5. 分隔符使用含歧义的符号（如 | 或 -）

## 五、返回导航 (BackNavigation)

### 5.1 组件概述

返回导航是用户返回上一级页面的交互机制，包含系统返回手势、顶部返回按钮、浏览器返回按钮等多种形态。

### 5.2 通用规范

**结构描述：**
- 顶部返回按钮：左箭头+可选文字
- 系统返回手势：边缘滑动
- 浏览器返回：键盘Backspace/Alt+Left

**尺寸参数：**
| 参数 | 数值 |
|------|------|
| 返回箭头图标尺寸 | 24dp |
| 触控区域 | 44-48dp |
| 返回文字字号 | 16-17sp |
| 箭头与文字间距 | 4dp |
| 边缘手势触发区域 | 20dp |
| 边缘手势滑动距离 | 50%屏幕宽度(返回) |

**状态变体：**
| 状态 | 说明 |
|------|------|
| 可返回 | 箭头可见/手势可用 |
| 不可返回 | 首页/根页面，箭头隐藏 |
| 返回确认 | 表单编辑中弹窗确认 |

**无障碍：**
- 返回按钮须有 `aria-label="返回"` 语义
- 手势返回须有替代的按钮操作路径
- 返回确认弹窗须可被屏幕阅读器聚焦

### 5.3 9大OS风格差异对比表

### 5.4 设计禁忌

1. 多级返回时跳过中间层级导致用户迷失
2. 表单编辑页未提供返回确认机制
3. 右滑手势与水平滚动列表冲突未处理
4. 返回按钮触控区域小于44x44dp
5. 返回目标页面状态未做缓存恢复
