# 输入控件规范

## 1. 组件概述

输入控件是用户与界面进行交互的基础元素，用于接收用户操作、输入数据或触发系统行为。本规范涵盖按钮、文本输入框、开关、滑块、复选框/单选框及下拉选择六大核心输入控件，确保跨平台设计的一致性与可用性。

## 2. 通用规范

### 2.1 结构描述

所有输入控件遵循统一的结构分层：
- **容器层**：定义控件外边界与基础尺寸
- **内容层**：承载文本、图标或交互标记
- **状态层**：通过颜色、阴影、描边变化反馈交互状态
- **辅助层**：提示文本、错误信息、加载指示器

### 2.2 尺寸参数

| 控件类型 | 最小尺寸 | 标准尺寸 | 最大尺寸 | 圆角范围 |
|---------|---------|---------|---------|---------|
| 按钮 | 32dp | 44dp | 56dp | 4-20dp |
| 文本输入框 | 40dp | 44dp | 48dp | 4-10dp |
| 开关 | 40x20dp | 48x24dp | 56x32dp | 全圆角 |
| 滑块 | 轨道4dp | 轨道6dp | 轨道8dp | 轨道全圆角 |
| 复选框/单选框 | 18dp | 20dp | 24dp | 复选框4dp/单选框全圆角 |
| 下拉选择 | 40dp | 48dp | 56dp | 4-12dp |

### 2.3 状态变体

| 状态 | 视觉表现 | 适用控件 |
|-----|---------|---------|
| 默认 | 基础色值、常规描边或填充 | 全部 |
| 悬停 | 亮度提升8%、轻微阴影增强 | 按钮、下拉选择 |
| 按下 | 亮度降低12%、内阴影或缩放0.98 | 按钮、开关、滑块 |
| 聚焦 | 2dp主色描边或外发光 | 文本输入框、下拉选择 |
| 禁用 | 透明度0.38、灰度化 | 全部 |
| 加载 | 旋转指示器替代内容、禁用交互 | 按钮、下拉选择 |
| 错误 | 红色描边、错误提示文本 | 文本输入框、下拉选择 |
| 成功 | 绿色对勾标记 | 文本输入框 |

### 2.4 无障碍

- 所有输入控件必须支持屏幕阅读器，提供准确的语义化标签（aria-label/role）
- 按钮最小触控区域不低于44x44dp，文本输入框触控区域不低于48dp高度
- 开关、复选框、单选框必须支持键盘操作（Tab聚焦、Space切换）
- 滑块必须支持方向键微调，步进值明确告知
- 颜色不可作为唯一信息传递手段，必须配合图标或文本
- 焦点指示器必须清晰可见，对比度不低于3:1

## 4. 设计禁忌

1. **禁止触控区域过小**：任何可点击输入控件的实际触控区域不得低于44x44dp，视觉上较小的控件需扩展透明触控热区
2. **禁止状态反馈缺失**：按钮按下、开关切换、滑块拖动必须有明确的即时视觉反馈，延迟不得超过100ms
3. **禁止颜色作为唯一状态标识**：色盲用户无法区分红绿，错误状态必须配合图标或文本说明
4. **禁止输入框无边距**：文本输入框内文字与边框距离不得低于12dp，避免视觉拥挤和误触
5. **禁止滑块步进值不明确**：有步进限制的滑块必须在视觉上标示刻度或当前数值，用户不可盲猜
6. **禁止开关与复选框混用**：开关用于即时生效的独立设置，复选框用于表单多选或批量操作，二者语义不可互换
7. **禁止下拉菜单层级过深**：下拉选择展开的菜单层级不得超过2级，超过时应改用页面导航或级联选择器
8. **禁用状态必须完全阻断交互**：禁用控件不仅要视觉灰显，还必须移除所有点击、聚焦、悬停事件响应
9. **禁止加载状态无进度提示**：异步操作中的按钮加载状态必须配合加载指示器，时长超过3s需追加文字说明
10. **禁止单选框单独出现**：单选框必须成组使用，且默认选中一项，禁止全部未选状态
