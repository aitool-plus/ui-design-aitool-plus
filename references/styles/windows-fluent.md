## 1. 设计哲学

### 1.1 五大核心支柱

| 支柱 | 英文 | 含义 |
|------|------|------|
| 光感 | Light | 光线引导注意力，强调交互焦点与层级关系 |
| 深度 | Depth | 通过Z轴层级、阴影、模糊建立空间秩序 |
| 动效 | Motion | 有意义的过渡与微交互，连接用户意图与界面响应 |
| 材质 | Material | Acrylic(亚克力)与Mica(云母)材质赋予界面物理质感 |
| 缩放 | Scale | 从微观图标到宏观窗口的统一设计语言跨越 |

### 1.2 四大设计原则

- 清晰 (Clarity)：信息层级分明，去除冗余装饰，聚焦内容本身
- 灵活 (Flexibility)：适配多种设备形态(桌面/平板/Xbox/HoloLens)，响应式布局
- 吸引 (Magnetism)：通过光感与材质创造沉浸式体验，引导用户探索
- 一致 (Consistency)：跨应用、跨设备保持统一的交互模式与视觉语言

### 1.3 技术组件体系

| 框架 | 说明 |
|------|------|
| WinUI 3 | 最新原生UI框架，支持Windows App SDK，独立于系统版本迭代 |
| Windows App SDK | 统一API层，解耦系统依赖，支持Win10 1809+ |
| Fluent UI React v9 | Web端Fluent Design实现，用于React应用 |
| Fluent UI Apple | macOS/iOS平台的Fluent Design适配 |

## 10. 设计禁忌

| 编号 | 禁忌项 | 说明 |
|------|--------|------|
| 1 | 禁止使用非Segoe UI Variable字体作为UI字体 | Windows 11统一使用Segoe UI Variable，旧版Segoe UI已弃用 |
| 2 | 禁止在WinUI控件上使用自定义圆角 | 必须遵循Fluent圆角规范(4/8/12/50%)，不得随意设置非标圆角 |
| 3 | 禁止使用iOS/Android风格的底部TabBar | Windows使用侧边NavigationView，底部TabBar不符合桌面端交互习惯 |
| 4 | 禁止使用Material Design的elevation阴影体系 | Fluent使用Light/Depth/Material三大支柱，elevation体系与Fluent设计语言冲突 |
| 5 | 禁止忽略窗口缩放和DPI适配 | 必须支持100%/125%/150%/200% DPI缩放，使用有效像素(epx)布局 |
| 6 | 禁止在对话框中使用非标准按钮布局 | ContentDialog按钮必须水平排列，主操作在右侧，遵循Windows交互惯例 |
| 7 | 禁止使用纯黑#000000作为深色模式背景 | 深色模式背景使用#202020，纯黑导致OLED烧屏且视觉过于生硬 |
