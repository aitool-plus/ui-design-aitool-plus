## 1. 设计哲学

### 1.1 四大核心原则

| 原则 | 英文 | 含义 |
|------|------|------|
| 友好 | Friendly | 界面亲切自然，操作流畅顺滑，降低用户认知负担 |
| 一致 | Consistent | 跨平台(iOS/Android)视觉与交互完全统一，WeUI组件库为唯一标准 |
| 清晰 | Clear | 信息层级分明，内容可读性强，操作路径简洁 |
| 高效 | Efficient | 减少操作步骤，提供即时反馈，优化高频场景效率 |

### 1.2 跨OS统一体验

- 无论iOS还是Android，小程序视觉和交互完全一致
- 不使用任何平台原生组件，全部使用WeUI渲染
- 导航栏、TabBar、弹窗等系统级UI由微信框架统一提供
- 用户无需适应不同平台的交互差异

### 1.3 技术组件体系

| 框架 | 说明 |
|------|------|
| WeUI | 微信官方设计团队提供的组件库，为小程序唯一标准UI库 |
| WeUI-WXSS | WeUI的WXSS样式库，提供基础样式变量与组件样式 |
| WeUI-Miniapp | 小程序版本的WeUI组件，支持自定义组件方式使用 |

## 6. 阴影与层级

### 6.1 阴影层级

| 层级名称 | 阴影值 | 适用场景 |
|----------|--------|----------|
| 卡片 | 0 1px 4px rgba(0,0,0,0.08) | 卡片轻微浮起效果 |
| 弹窗 | 0 8px 24px rgba(0,0,0,0.16) | 模态弹窗、ActionSheet |

### 6.2 层级规则

- 微信小程序无elevation体系，仅卡片和弹窗使用阴影
- 层级通过背景色深浅和分隔线区分
- 列表项不使用阴影，依靠背景色与间距区分层级
- 页面层级不超过5级，防止导航栈过深

## 7. 动效规范

### 7.1 时长规范

| 动效类型 | 时长 | 缓动曲线 | 适用场景 |
|----------|------|----------|----------|
| 组件动画 | 200ms | ease | 按钮、开关、复选框状态切换 |
| 页面转场 | 300ms | ease-in-out | 页面推入/推出 |
| 弹窗动画 | 300ms | ease-in-out | 模态弹窗弹出/收起 |
| 下拉刷新 | 自定义 | - | 系统动画，可自定义样式 |

### 7.2 动效约束

- 禁止自定义页面转场动画，必须使用微信默认推入/推出
- 禁止使用过长的动画时长(超过500ms)
- 禁止在列表滚动时触发同步动画(会造成卡顿)
- 建议使用CSS transition替代JS动画，性能更优

### 7.3 微交互规范

| 交互 | 反馈方式 |
|------|----------|
| 按钮点击 | 透明度降低至0.7 |
| 列表项点击 | 背景色变为#ECECEC |
| 开关切换 | 200ms平滑过渡 |
| 下拉刷新 | 系统提供的自定义动画 |
| 加载中 | WeUI Loadmore组件 |

## 10. 设计禁忌

| 编号 | 禁忌项 | 说明 |
|------|--------|------|
| 1 | 禁止使用非WeUI标准组件 | 必须使用WeUI组件库，自定义组件需与WeUI视觉完全一致 |
| 2 | 禁止自定义导航栏样式 | 除非设置navigationStyle: custom，否则必须使用系统默认导航栏 |
| 3 | 禁止在TabBar使用非标准图标 | TabBar图标必须使用24px线性/填充风格，颜色遵循选中/未选中规范 |
| 4 | 禁止使用iOS/Android原生组件替代WeUI组件 | 小程序追求跨平台一致体验，必须使用WeUI渲染 |
| 5 | 禁止使用微信绿以外的颜色作为主操作色 | 除非品牌定制场景，主操作必须使用#07C160 |
| 6 | 禁止页面层级超过5级 | 导航栈过深影响用户体验，且返回操作复杂度急剧上升 |
| 7 | 禁止使用自定义页面转场动画 | 页面转场由微信框架统一控制，自定义会破坏体验一致性 |
| 8 | 禁止在深色模式下使用纯黑#000000作为背景 | 深色模式背景使用#111111，纯黑视觉过于生硬 |
| 9 | 禁止列表项无点击反馈 | 列表项点击必须有背景色变化(#ECECEC)，否则用户无法感知操作 |
| 10 | 禁止使用过小点击区域 | 可点击元素最小点击区域44x44px，低于此值无法保证操作准确性 |

| 参数 | 数值 (px) | 说明 |
|------|----------|------|
| TabBar高度 | 50px | 不含安全区 |

| 底部安全区 | 34px(iPhone X+) | Home Indicator区域 |

### 11.4 刘海屏/挖孔屏适配规则

### 11.5 折叠屏适配规则

- 小程序自适应折叠屏宽度变化

- 折叠屏展开时需重新计算布局

## 15. 性能与渲染约束

### 15.1 动画帧率要求

| 场景 | 帧率要求 | 说明 |
|------|---------|------|
| 核心交互动画 | 60fps | 不得低于60fps |
| 页面转场 | 60fps | 系统动画 |
| 滚动 | 60fps | 列表流畅 |

### 15.2 模糊效果性能建议

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 高斯模糊半径上限 | 20px | 超过性能下降 |
| 推荐模糊半径 | 15px | 轻模糊 |
| 实时模糊区域 | 不超过屏幕20% | 大面积模糊避免使用 |
| 替代方案 | 半透明背景色 | 低端设备降级 |

### 15.3 阴影渲染建议

| 建议 | 说明 |
|------|------|
| 卡片阴影 | 仅使用0 1px 4px rgba(0,0,0,0.08) |
| 弹窗阴影 | 仅使用0 8px 24px rgba(0,0,0,0.16) |
| 列表项 | 不使用阴影 |
| 阴影数量 | 单屏动态阴影不超过3个 |

### 15.4 图片加载策略

| 加载阶段 | 策略 | 说明 |
|---------|------|------|
| 占位色 | 主色调低饱和度色块 | 即时展示 |
| 低清图 | 缩略图(宽200px) | 优先加载 |
| 高清图 | 原图 | 后台加载替换 |
| 缓存 | wx.getImageInfo+本地缓存 | 小程序缓存 |
| 懒加载 | lazy-load属性 | 图片懒加载 |

### 15.5 列表虚拟滚动建议

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 长列表阈值 | 100条以上 | 使用recycle-view |
| 分页加载 | 每页20-50条 | onReachBottom |
| 列表项复用 | recycle-view | 自定义组件复用 |
| 避免setData过频 | 20ms间隔以上 | 防止渲染阻塞 |

### 15.6 重绘优化建议

| 规则 | 说明 |
|------|------|
| 减少setData频率 | 合并更新，避免频繁调用 |
| 避免大对象setData | 仅传输变更数据 |
| 减少节点数 | 单个页面节点不超过1000个 |
| 避免不必要的WXS | 复杂计算使用WXS替代JS |
| 图片优化 | 使用WebP格式，压缩质量80% |
