## 1. 设计哲学

### 1.1 三大核心原则

| 原则 | 英文 | 含义 |
|------|------|------|
| 万物互联 | Seamless Connection | 跨设备服务流转，手机/平板/手表/车机/智慧屏无缝协同 |
| 服务流转 | Service Flow | 服务随用户跨设备流转，不依赖特定设备，以人为中心 |
| 弹性动效 | Elastic Motion | 弹簧物理模型驱动动效，自然流畅，符合物理直觉 |

### 1.2 设计准则

- 服务卡片化：核心服务以卡片形式呈现，信息前置，减少操作层级
- 分布式能力：应用可跨设备部署，UI自适应不同屏幕形态
- 弹性设计：组件尺寸、间距、字号弹性适配，支持1.5x-3x缩放
- 全场景协同：超级终端、多屏协同、智慧跨端体验统一

### 6.2 服务卡片悬浮阴影

| 卡片状态 | 阴影参数 | 说明 |
|---------|---------|------|
| 静态 | 0vp 2vp 8vp rgba(0,0,0,0.08) | 默认展示 |

### 6.3 深色模式阴影调整

- 深色模式下阴影效果减弱，主要依赖亮度差区分层级
- 深色模式阴影色值：rgba(0,0,0,0.30) ~ rgba(0,0,0,0.60)
- 服务卡片深色模式使用边框发光替代阴影：1vp rgba(255,255,255,0.08)

## 10. 设计禁忌

### 10.1 绝对禁止

1. **禁止静态卡片**：服务卡片必须包含交互能力或实时信息更新，不得设计为纯装饰静态卡片
2. **禁止孤立设计**：组件设计必须考虑跨设备流转场景，不得仅针对单一设备设计
3. **禁止使用非HarmonyOS Sans字体**：UI界面必须使用HarmonyOS Sans，品牌展示区除外
4. **禁止打破弹性布局**：组件尺寸必须支持弹性缩放，不得使用固定像素值
5. **禁止忽略服务流转**：核心功能必须支持跨设备流转，不得绑定单一设备
6. **禁止自定义系统控件**：Switch、Slider、Progress等系统控件不得替换为自定义实现
7. **禁止使用固定色值代替语义色**：必须使用系统语义化颜色，不得硬编码色值
8. **禁止忽略深色模式**：所有界面必须同时设计浅色/深色两套方案

### 10.2 强烈不建议

1. 不建议在深色模式下使用纯白(#FFFFFF)背景
2. 不建议使用小于44vp的触控热区
3. 不建议使用线性动画替代弹性动画
4. 不建议在服务卡片中使用过多文字信息
5. 不建议忽略跨设备布局适配
6. 不建议在弹窗中使用关闭按钮替代手势关闭
7. 不建议忽略无障碍对比度要求（文字与背景对比度不低于4.5:1）
8. 不建议在卡片中使用超过3级信息层级

## 15. 性能与渲染约束

### 15.1 动画帧率要求

| 场景 | 帧率要求 | 说明 |
|------|---------|------|
| 核心交互动画 | 60fps最低 | 不得低于60fps |
| 页面转场 | 60fps | 弹性动画 |
| 滚动 | 60fps | List保证流畅 |
| 服务卡片动画 | 60fps | 卡片展开/收起 |

### 15.2 模糊效果性能建议

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 高斯模糊半径上限 | 25vp | 超过25vp性能下降 |
| 推荐模糊半径 | 20vp | 系统默认 |
| 实时模糊区域 | 不超过屏幕30% | 大面积模糊降级 |
| 替代方案 | 半透明背景色 | 低端设备降级 |

### 15.3 阴影渲染建议

| 建议 | 说明 |
|------|------|
| 静态阴影 | 使用elevation属性 |
| 动态阴影 | 状态切换管理 |
| 深色模式 | 亮度差替代阴影 |
| 阴影数量 | 单屏动态阴影不超过6个 |

### 15.4 图片加载策略

| 加载阶段 | 策略 | 说明 |
|---------|------|------|
| 占位色 | 主色调低饱和度色块 | 即时展示 |
| 低清图 | 缩略图(宽200px) | 优先加载 |
| 高清图 | 原图 | 后台加载替换 |
| 缓存 | 双缓存 | 内存+磁盘 |

### 15.5 列表虚拟滚动建议

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 长列表阈值 | 50条以上 | 必须使用LazyForEach |
| 缓存池 | 可见项+2 | 组件复用 |
| 分页加载 | 每页20-50条 | 滚动到底自动加载 |

### 15.6 重绘优化建议

| 规则 | 说明 |
|------|------|
| 避免布局抖动 | 禁止渲染中修改布局 |
| 减少视图层级 | 层级不超过6层 |
| 避免过度绘制 | 背景重叠不超过2.5x |
| 懒加载 | 不可见区域延迟加载 |
