# Apple iOS Human Interface Guidelines 设计规范

> @author aitool.plus

## 1. 设计哲学

### 1.1 三大核心原则

| 原则 | 英文 | 含义 |
|------|------|------|
| 清晰 | Clarity | 文字易读、图标精确、装饰恰当、聚焦功能 |
| 遵从 | Deference | 流畅的动效引导、UI服务于内容而非喧宾夺主 |
| 深度 | Depth | 层级关系通过动效和模糊清晰表达，增强理解 |

### 1.2 设计准则

- 一致性：系统级交互模式统一，遵循用户已有心智模型
- 直接操作：用户直接触摸、滑动、捏合操作内容
- 隐喻：使用现实世界的隐喻降低认知成本
- 反馈：每一次操作都有即时视觉/触觉反馈
- 用户控制：用户始终拥有主动权，可随时中断或撤销

### 5.2 连续性圆角 (Continuous Corners)

- iOS使用连续性圆角而非标准圆角，曲线更接近超椭圆
- 核心公式：使用三次贝塞尔曲线近似超椭圆，控制点偏移量约为圆角半径的0.552倍
- CSS近似：`border-radius: 16pt;` 配合 `corner-smoothing` 属性

### 6.2 模糊效果 (Blur / Material)

| 模糊类型 | 参数 | 用途 |
|---------|------|------|
| Ultra Thin Material | 模糊20pt + 饱和度85% + 亮度0.99 | 极轻透明 |
| Thin Material | 模糊20pt + 饱和度85% + 亮度0.95 | 轻透明 |

### 6.3 深色模式阴影调整

- 深色模式下阴影效果减弱，主要依赖亮度差和模糊层区分层级
- 深色模式阴影色值：rgba(0,0,0,0.25) ~ rgba(0,0,0,0.50)
- 深色模式模糊亮度降低：0.40 ~ 0.80

## 7. 动效规范

### 7.1 动画时长

| 动画类型 | 时长 (ms) | 说明 |
|---------|----------|------|
| 微交互 (Tap/Press) | 100-200 | 按钮点击、开关切换 |
| 标准转场 | 350 | 页面Push/Pop、模态弹出 |
| 大幅转场 | 500-700 | 共享元素转场、全屏展开 |
| 连续手势 | 实时 | 拖拽、滑动跟随 |
| 弹性回弹 | 600-800 | 滚动回弹、过度拖拽 |

### 7.2 缓动曲线 (Spring Animation)

| 场景 | 阻尼 (Damping) | 刚度 (Stiffness) | 质量 (Mass) | 说明 |
|------|---------------|-----------------|------------|------|
| 标准Spring | 0.85 | 200 | 1.0 | 通用弹性动画 |
| 轻柔Spring | 0.90 | 150 | 1.0 | 柔和过渡 |
| 活泼Spring | 0.70 | 250 | 1.0 | 活泼弹跳 |
| 惯性Spring | 0.95 | 300 | 1.0 | 快速响应 |
| 滚动减速 | 0.80 | 180 | 1.0 | 滚动自然减速 |

### 7.3 贝塞尔曲线 (非Spring场景)

| 场景 | 曲线参数 | 说明 |
|------|---------|------|
| EaseIn | cubic-bezier(0.42, 0, 1, 1) | 加速进入 |
| EaseOut | cubic-bezier(0, 0, 0.58, 1) | 减速退出 |
| EaseInOut | cubic-bezier(0.42, 0, 0.58, 1) | 标准缓入缓出 |
| iOS Default | cubic-bezier(0.25, 0.1, 0.25, 1) | iOS系统默认 |

### 7.4 转场规则

| 转场类型 | 动画描述 | 时长 |
|---------|---------|------|
| Push | 新页面从右侧滑入，当前页面向左推出 | 350ms |
| Pop | 当前页面向右滑出，下级页面从左侧滑入 | 350ms |
| Modal Present | 新页面从底部升起，带弹性效果 | 400ms |
| Modal Dismiss | 页面向下滑落消失 | 350ms |
| Flip | 页面3D翻转 | 600ms |
| Cross Dissolve | 淡入淡出 | 350ms |
| Shared Element | 共享元素平滑过渡到新位置 | 500-700ms |
| Large Title Collapse | 大标题收缩到导航栏 | 跟随滚动实时 |

### 7.5 触觉反馈 (Haptic)

| 反馈类型 | 触觉模式 | 用途 |
|---------|---------|------|
| Light Impact | 轻触 | 轻量操作反馈 |

## 10. 设计禁忌

### 10.1 绝对禁止

1. **禁止使用非系统字体**：不得在界面中使用SF Pro以外的字体作为UI字体，品牌展示区除外
2. **禁止自定义TabBar样式**：TabBar必须遵循系统标准，不得改变高度、图标尺寸、选中态样式
3. **禁止打破安全区**：内容不得侵入状态栏、Home Indicator区域
4. **禁止自定义系统控件外观**：Switch、Slider、Progress等系统控件不得替换为自定义实现
5. **禁止使用固定色值代替动态色**：必须使用系统语义化颜色(UIColor.label等)，不得硬编码色值
6. **禁止遮挡系统手势**：不得阻止边缘滑动返回、底部上拉控制中心等系统手势
7. **禁止在模态弹窗中使用关闭按钮**：系统模态通过下滑关闭，不得添加X按钮
8. **禁止非连续性圆角**：弹窗、卡片等必须使用连续性圆角，不得使用标准CSS圆角

### 10.2 强烈不建议

1. 不建议在导航栏使用自定义背景，应使用系统模糊效果
2. 不建议创建自定义刷新控件，应使用UIRefreshControl
3. 不建议在列表中使用非44pt触控高度的交互元素
4. 不建议在深色模式下使用纯白(#FFFFFF)背景
5. 不建议使用线性动画替代Spring动画
6. 不建议在TabBar中放置超过5个Tab
7. 不建议使用自定义返回按钮替代系统返回
8. 不建议忽略Dynamic Type适配
